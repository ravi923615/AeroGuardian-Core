# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` | Yes |

## Reporting a Vulnerability

Do not open a public issue for an unpatched vulnerability.

Use GitHub Private Vulnerability Reporting for this repository when it is available. If private reporting is unavailable, contact the repository maintainer through the email address associated with the repository owner account and include the following:

- A short description of the issue and the affected component.
- Reproduction steps or a proof of concept.
- The impact you observed or expect.
- Any suggested remediation, workaround, or guardrail.

For non-sensitive defects, feature requests, or documentation issues, use the standard issue tracker.

## Disclosure Expectations

- Initial acknowledgement target: within 5 business days.
- Triage and severity assessment target: within 10 business days.
- Coordinated disclosure is preferred after a fix or mitigation is available.
- Please avoid public disclosure until maintainers have had a reasonable opportunity to investigate and patch the issue.

## Scope Notes

- Findings should be actionable against code or configurations maintained in this repository.
- Third-party service outages or rate limits are out of scope unless they stem from a defect in this project.
- Research ideas without a demonstrable security impact are better handled as issues or discussions instead of vulnerability reports.
